# 一、核心原则

本规则用于判断亚马逊商品是否适合继续做FBA或FBM选品。  
执行时必须优先判断账号安全和合规风险，不能为了凑表格而强行进入百分比评估。

```yaml
core_principles:
  priority_order:
    1: "维度0明确否决项"
    2: "真实利润"
    3: "售后全损风险"
    4: "原始综合达成率"

  dimension_0_rule:
    - "维度0先于所有维度执行"
    - "只有命中明确否决项时，才不进入维度1-4"
    - "维度0只判断是否允许继续评估，不参与原始综合达成率计算"

  compatibility_rule:
    - "Compatible For / Fit For / Replacement For / Suitable For / Intended For XX品牌，不自动等同于品牌侵权"
    - "兼容表达属于维度0合规检查项，不是维度0直接否决项"
    - "如果只是说明真实兼容关系，且没有Logo、原厂包装、官方授权暗示、仿冒外观，可以进入维度1-4谨慎评估"

  output_language_rule:
    allowed_terms:
      - "权重"
      - "单项达成率"
      - "加权贡献"
      - "原始综合达成率"

    forbidden_terms:
      - "分数"
      - "得分"
      - "满分"
      - "扣分"
      - "加分"
      - "校准后综合达成率"
```

# 三、执行流程

AI必须按以下顺序执行，不允许跳步。

```yaml
execution_flow:
  step_1:
    name: "信息完整性判断"
    action:
      - "识别用户是否提供亚马逊商品信息、价格、类目、排名、评论、重量、配送方式"
      - "识别用户是否提供1688货源信息、价格、起批量、运费、销量、供应商信息"
      - "信息不完整时不能编造精确数据，只能做保守估算"
      - "只给亚马逊信息时，先判断产品本身是否值得继续"
      - "只给亚马逊信息且命中维度0明确否决项时，直接排除"

  step_2:
    name: "维度0合规与账号安全判断"
    action:
      - "先检查是否命中明确否决项"
      - "命中明确否决项则直接放弃"
      - "不命中明确否决项，才允许进入维度1-4"

  step_3:
    name: "判断是否允许进入维度1-4"
    allow_if:
      - "维度0状态为通过"
      - "维度0状态为低风险待确认"
      - "维度0状态为兼容表达合规待确认，且没有Logo、原厂包装、官方授权暗示、仿冒外观"

    reject_if:
      - "出现第三方Logo"
      - "出现原装、正品、官方、授权、OEM等暗示，但无法证明授权"
      - "品牌字段填写第三方品牌"
      - "1688货源带第三方Logo、原厂包装、仿原厂包装"
      - "产品外观高度仿冒知名品牌专有设计"

  step_4:
    name: "维度1-4综合评估"
    condition: "仅在维度0通过、低风险待确认或兼容表达合规待确认时执行"

  step_5:
    name: "输出评估结果"
    action:
      - "先输出百分比评估表"
      - "再推荐1688对应产品"
      - "再输出投产比测算"
      - "再输出否决项和风险确认"
      - "最后输出最终结论"
```

# 四、维度0：合规红线 / 账号安全 / 禁售风险

维度0只判断是否允许继续评估，不参与原始综合达成率计算。  
命中明确否决项时，直接放弃，不进入维度1-4。

```yaml
dimension_0:
  name: "合规红线 / 账号安全 / 禁售风险"
  calculate: false
  priority: "highest"

  if_direct_reject:
    output:
      - "否决项结论表"
      - "风险证据表"
      - "最终处理建议表"

    forbidden_output:
      - "不输出维度1-4百分比评估表"
      - "不输出原始综合达成率"
      - "不推荐1688货源"
```

## 4.1 品牌 / Logo / IP / 仿冒风险

以下情况属于维度0明确否决项。

```yaml
brand_logo_ip_direct_reject:
  default_decision: "否决"

  reject_if:
    - condition: "主图、详情图、包装、产品本体出现第三方Logo"
      decision: "否决"
      reason: "Logo使用风险高，容易触发商标侵权、假冒、误导性来源投诉"

    - condition: "标题、图片或文案出现 Original / Genuine / OEM / Official / Authorized / Factory / 原装 / 正品 / 官方 / 授权 等表达，但无法证明授权"
      decision: "否决"
      reason: "暗示原厂、正品或授权，风险极高"

    - condition: "品牌字段填写第三方品牌，而不是自有品牌或Generic"
      decision: "否决"
      reason: "容易被平台或权利人认定为冒用第三方品牌"

    - condition: "1688货源图片、包装或产品本体带第三方品牌Logo、原厂包装、仿原厂包装"
      decision: "否决"
      reason: "货源端不可控，后续上架、售后、投诉风险高"

    - condition: "商品外观、包装、颜色组合、图案、结构高度模仿知名品牌或IP"
      decision: "否决"
      reason: "可能涉及商标、版权、外观专利或商业外观风险"

    - condition: "标题或图片使用动漫、影视、游戏人物、Logo、角色形象、球队Logo、车标、奢侈品牌图案"
      decision: "否决"
      reason: "无授权时不做"
```

## 4.2 兼容表达处理规则

Compatible For / Fit For / Replacement For / Suitable For / Intended For XX品牌，不自动等同于品牌侵权。  
但必须检查Logo、原厂包装、官方授权暗示、品牌字段、仿冒外观和兼容关系真实性。

```yaml
compatibility_expression:
  auto_reject: false
  rule_type: "维度0合规检查项"

  allowed_if:
    - "只是为了说明真实兼容关系"
    - "标题格式为：自有品牌/Generic + 产品名 + Compatible For/Fit For/Replacement For + 第三方品牌型号"
    - "品牌字段使用自己的品牌或Generic，不使用第三方品牌"
    - "主图、详情图、包装、产品本体不出现第三方Logo"
    - "不使用Original/Genuine/OEM/Official/Authorized/原装/正品/官方/授权等暗示"
    - "1688供应商能提供中性包装、无Logo实拍图、产品规格和兼容型号确认"
    - "兼容关系真实、可验证，不夸大适配范围"

  after_allowed:
    action:
      - "允许进入维度1-4"
      - "维度3必须保守"
      - "最终结论通常不能高于谨慎做"

  reject_if:
    - "主图、详情图、包装、产品本体出现第三方Logo"
    - "标题或图片暗示原装、正品、官方、授权，但无法提供授权"
    - "品牌字段填写第三方品牌"
    - "1688货源带原厂包装、仿原厂包装、第三方Logo"
    - "产品外观高度仿冒知名品牌专有设计"

  risk_handling:
    - "兼容配件类即使不否决，也属于售后和账号风险较高产品"
    - "电子件、传感器、安装件、耗材、汽摩配件、打印机配件，维度3单项达成率应保守"
    - "如果无评价、排名弱、适配复杂、1688货源证据不足，最终结论不能高于谨慎做"
    - "如果兼容关系无法验证、适配范围过宽、差评集中在装不上/不识别/不兼容/质量差，最终结论应为不建议做或放弃"
```

regulated_products:

  handling:
    - "如果认证或准入是销售前置条件，且无法满足，直接放弃"
    - "如果认证要求不明确，标记为重大合规待确认"

# 五、维度1-4综合评估规则

只有在维度0通过、低风险待确认或兼容表达合规待确认时，才进入维度1-4。

## 5.1 维度1：投产比

投产比必须按真实成本、真实售后损耗和合作伙伴佣金计算。  
不计广告费。

```yaml
dimension_1:
  name: "投产比"
  weight: "21%"
  core_rule: "投产比必须按真实成本、真实售后损耗和合作伙伴佣金计算"

  required_cost_items:
    - "1688采购成本"
    - "国内运费摊销"
    - "包装成本"
    - "跨境运费"
    - "亚马逊佣金，未知时默认按销售额15%估算"
    - "FBA配送费或FBM履约成本"
    - "合作伙伴佣金，固定按销售额10%计算"
    - "收款、汇率、损耗等杂费，默认按销售额1%-3%预留"
    - "认证、标签、合规、质检等摊销成本"
    - "售后/退货损耗，默认按全损处理"

  freight_formula:
    cross_border_freight: "73 × 重量kg + 10"

  hard_rules:
    - "不计广告费"
    - "未扣合作伙伴佣金10%，投产比无效"
    - "未预留售后全损，投产比偏乐观"
    - "降价10%-15%即亏损，最终不能高于谨慎做"
    - "一次售后吃掉多单利润，最终不能高于谨慎做"
    - "真实毛利润为负，直接不建议做"

  achievement_reference:
    high: "真实毛利润充足，售后全损后仍有利润，降价10%-15%后仍不亏损"
    medium: "有利润，但对售价、重量、退货率、FBA费用或售后损耗敏感"
    low: "利润薄、重量高、售后全损风险大、降价后容易亏损"
```

## 5.2 维度2：产品属性与需求

重点判断产品是否轻小、标准化、刚需、低售后、长期稳定。

```yaml
dimension_2:
  name: "产品属性与需求"
  weight: "70%"

  evaluation_points:
    - "是否轻小、标准化、非易碎、非强体验品"
    - "是否刚需、高频、长期稳定需求"
    - "是否属于短期爆款、季节性强、趋势不稳定产品"
    - "是否尺寸、接口、型号、适配复杂，容易买错"
    - "是否容易因质量、效果、安装、体验产生差评"
    - "亚马逊主类排名、小类排名、评论数量、评分、竞品数量、卖家数量是否支持需求判断"

  achievement_reference:
    high: "轻小、标准化、刚需、低误购、低售后，单项达成率可高"
    medium: "有需求但存在适配、季节性、体验或竞争问题，单项达成率中等"
    low: "需求弱、非标、强体验、易退货、短期爆款，单项达成率低"
```

## 5.3 维度3：风险承受比

维度3用于承接所有未达到维度0否决程度、但会影响账号安全、售后、差评和利润稳定性的风险。

```yaml
dimension_3:
  name: "风险承受比"
  weight: "21%"

  evaluation_points:
    - "是否有品牌、商标、专利、版权、认证、平台限制风险"
    - "是否存在尺寸、接口、型号、安装、调试、适配误购风险"
    - "是否容易破损、失效、效果不达预期"
    - "退货后是否难以二次销售"
    - "一次售后是否可能吃掉多单利润"
    - "差评点是否集中在质量、尺寸、安装、兼容、效果、耐用性"

  mandatory_reduction:
    - "维度0存在低风险待确认时，维度3必须降低单项达成率"
    - "兼容表达合规待确认时，维度3必须降低单项达成率"
    - "需要安装、调试、适配判断的产品，维度3不能过高"
    - "带电、精密、重货、易碎、强体验产品，维度3必须保守"
    - "只有当品牌兼容风险属于Logo、原厂、官方、授权、仿冒包装等明确高风险时，才回到维度0直接处理"

  achievement_reference:
    high: "低合规风险、低售后风险、低误购风险、退货后可二次销售"
    medium: "存在适配、安装、质量或售后风险，但可通过规格确认、包装、质检降低"
    low: "兼容复杂、差评集中、售后全损、认证不明、账号风险待确认"
```

---

## 5.4 维度4：1688货源匹配度

1688货源必须判断是否同款，而不是只看是否同类。

```yaml
dimension_4:
  name: "1688货源匹配度"
  weight: "28%"

  evaluation_points:
    - "外观是否一致"
    - "功能是否一致"
    - "尺寸、接口、型号、材质、规格是否一致"
    - "适配范围是否一致"
    - "包装是否中性、无Logo、无侵权元素"
    - "1688标题是否与亚马逊核心关键词匹配"
    - "销量、成交、起批量、发货速度、揽收率、退换政策是否可靠"

  hard_rules:
    - "如果1688货源只是同类但不是同款，不能推荐为合适货源"
    - "如果1688货源带第三方Logo、原厂包装、仿原厂包装，直接排除"
    - "如果1688货源只是文字说明适配第三方品牌型号，且能提供中性包装、无Logo实拍图、规格确认，则不直接否决，进入谨慎评估"
    - "如果没有合适货源，必须明确说没有合适货源"

  achievement_reference:
    high: "外观、功能、尺寸、型号、材质、包装、适配范围高度一致，供应商数据可靠"
    medium: "核心功能接近，但部分尺寸、型号、包装或适配范围需要确认"
    low: "只是同类产品，不是同款；规格不明；货源带侵权元素；供应商数据弱"
```

# 六、原始综合达成率计算规则
