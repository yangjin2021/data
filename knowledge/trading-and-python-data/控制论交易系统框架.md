# 控制论交易系统框架

> 目标：把控制论、信息论、最优控制、质量控制、灵敏度理论和模糊判断，用于构建一个不失控、可反馈、可调节、可复盘的交易系统。

## 一句话原则

交易系统不是预测机器，而是风险受控的反馈调节系统。

交易控制论的目标不是每一笔都赚钱，而是：

```text
长期不爆仓
回撤可控
仓位可控
亏损可停
参数可调
结果可复盘
系统可进化
```

---

## 一、把账户看成被控系统

### 被控对象

```text

```

状态变量不能只看账户总数，还要拆成账户级状态、持仓级状态、币种结构状态和交易行为状态。

## 二、交易系统的控制闭环

```text
读取市场信号
  ↓
估计账户状态
  ↓
估计每个持仓状态
  ↓
估计币种结构风险
  ↓
判断风险偏差
  ↓
生成控制动作
  ↓
执行或拒绝交易
  ↓
记录结果
  ↓
反馈修正参数
```

---

## 三、第一层：硬边界控制

硬边界是系统的安全阀，任何策略不得突破。

### 建议参数

```text
最大杠杆 ≤ 5x
单笔开仓金额 ≤ 可用余额 5%
最大同时持仓数 ≤ 2
单个币种仓位 ≤ 账户权益 5%
全部山寨币总仓位 ≤ 账户权益 10%
单日最大亏损 ≤ 账户权益 2%
总回撤超过 5% 后暂停开仓
连续亏损 3 笔后暂停交易
持仓时间超过计划时间后必须复核
高波动异常时只允许平仓，不允许开新仓
```

### 作用

```text
防爆仓
防重仓
防单币种失控
防持仓时间过长变成被动扛单
防连续失控
防情绪化加仓
防高波动乱交易
```

## 四、第二层：信号处理

交易系统接收到的是信号，不是真相。

### 原始信号

```text
价格变化
成交量变化
资金费率
波动率
K线形态
订单簿深度
市场情绪
```

### 处理目标

```text
去噪
确认趋势
识别震荡
发现异常
判断流动性
估计风险强度
```

### 原则

```text
单一信号不能下结论
多个信号同向才提高可信度
信号冲突时降低仓位或不交易
```

## 五、第三层：选币控制

选币不是找“必涨币”，而是筛掉不适合交易的币。

### 选币过滤条件

```text
成交额足够大
流动性足够好
点差不能过大
波动不能过度异常
资金费率不能极端
近期不能处于剧烈拉盘或砸盘后
不能与已有持仓高度同向相关
不能让某一类币种风险过度集中
```

### 选币评分维度

```text
流动性分
趋势强度分
波动风险分
资金费率风险分
成交量确认分
异常风险分
与当前持仓相关性风险分
持仓结构贡献分
```

### 输出格式

```text
可交易
只观察
跳过
禁止交易
```

## 六、第四层：方向判断控制

方向判断不能只靠感觉。

### 多头条件示例

```text
趋势偏多
成交量支持上涨
回调不破关键结构
波动没有失控
资金费率不过热
已有持仓结构允许继续增加多头风险
```

### 空头条件示例

```text
趋势偏空
反弹无力
成交量支持下跌
关键支撑失守
资金费率不过度拥挤
已有持仓结构允许继续增加空头风险
```

### 不交易条件

```text
方向冲突
波动过大
流动性差
资金费率极端
连续亏损后
账户处于暂停状态
当前持仓币种过多
当前同方向风险过度集中
持仓时间过长且未复核
```

---

## 七、第五层：仓位调节

仓位是控制变量，不是情绪表达。

### 基础规则

```text
普通状态：单笔 ≤ 余额 5%
连续亏损 1 笔：降到 3%
连续亏损 2 笔：降到 2%
连续亏损 3 笔：暂停开仓
高波动状态：仓位减半
低确定性信号：只观察或最小仓位
持仓时间超过计划：禁止加仓，只允许复核、减仓或平仓
单币浮亏扩大：禁止同币种加仓
多币种同向亏损：降低整体方向暴露
```

### 控制论解释

```text
亏损反馈 = 系统偏离目标
持仓时间过长 = 控制动作未按预期收敛
降低仓位 = 减小控制强度
暂停交易 = 防止系统发散
```

---

## 八、第六层：杠杆调节

杠杆是放大器，也是灵敏度放大器。

### 原则

```text
最大 5x
高波动时 ≤ 3x
连续亏损后 ≤ 2x
信号冲突时不用杠杆或不交易
持仓时间越长，允许杠杆越低
浮亏扩大时禁止提高杠杆
```

### 控制论解释

```text
杠杆越高，系统对价格扰动越敏感。
持仓时间越长，不确定性累积越大。
灵敏度过高的系统更容易失控。
```

---

## 九、第七层：持仓生命周期控制

持仓不是静态对象，而是有生命周期的动态系统。

### 每个持仓必须定义

```text
开仓时间
计划持仓时间
最大允许持仓时间
入场理由
无效条件
止损位置
止盈条件
减仓条件
强制复核时间
异常退出条件
```

### 持仓时间控制

```text
短线交易：超过计划时间仍未盈利，必须复核
浮亏持仓：持仓时间越长，风险等级越高
浮盈持仓：监控回吐比例，防止盈利变亏损
超时持仓：禁止加仓，只允许减仓、平仓或重新确认逻辑
```

### 持仓盈亏控制

```text
单币浮亏达到阈值：触发止损或减仓
单币浮盈达到阈值：触发移动止盈或分批止盈
最大浮盈回吐超过阈值：触发保护性平仓
多个币种同时亏损：说明系统性方向判断可能错误
```

## 十、第八层：止损、止盈与最优停止

最优停止回答：什么时候该停。

### 必须定义

```text
入场理由
无效条件
止损位置
止盈条件
最大持仓时间
异常退出条件
```

### 停止条件

```text
达到止损
达到止盈
入场逻辑失效
持仓时间超过计划
最大浮盈明显回吐
波动异常扩大
连续亏损触发暂停
账户回撤超过阈值
```

## 十二、第十层：自适应学习

交易系统要根据反馈调节参数。

### 参数调节示例

```text
连续亏损 → 降仓或暂停
胜率下降 → 降低交易频率
盈亏比下降 → 优化止损止盈
平均持仓时间变长 → 缩短策略有效期或加强超时退出
浮盈回吐过多 → 增加移动止盈
某类币种持续亏损 → 降低该类币种权重或禁止交易
高波动亏损多 → 增加波动过滤
震荡行情亏损多 → 增加趋势过滤
```

### 目标

```text
不是让系统越来越激进，而是让系统越来越稳健。
```

## 十四、最重要的一句话

交易系统不是为了预测未来，而是为了在未来不可预测的情况下，让账户、仓位、持币时间、币种结构和盈亏状态都不失控。
