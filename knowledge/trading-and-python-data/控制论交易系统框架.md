# 控制论交易系统框架

> 目标：把控制论、信息论、最优控制、质量控制、灵敏度理论和模糊判断，用于构建一个不失控、可反馈、可调节、可复盘的交易系统。

## 一句话原则

交易系统不是预测机器，而是风险受控的反馈调节系统。

交易控制论的目标不是每一笔都赚钱，而是：

```text
长期不爆仓
回撤可控
仓位可控
亏损可停
参数可调
结果可复盘
系统可进化
```

## 二、交易系统的控制闭环


## 三、第一层：硬边界控制

硬边界是系统的安全阀，任何策略不得突破。

### 建议参数

```text
最大杠杆 ≤ 5x
单笔开仓金额 ≤ 可用余额 5%
最大同时持仓数 ≤ 2
单日最大亏损 ≤ 账户权益 2%
总回撤超过 5% 后暂停开仓
连续亏损 3 笔后暂停交易
高波动异常时只允许平仓，不允许开新仓
```

### 作用

```text
防爆仓
防重仓
防连续失控
防情绪化加仓
防高波动乱交易
```

## 四、第二层：信号处理

交易系统接收到的是信号，不是真相。

### 原始信号

```text
价格变化
成交量变化
资金费率
波动率
K线形态
订单簿深度
市场情绪
```

### 处理目标

```text
去噪
确认趋势
识别震荡
发现异常
判断流动性
估计风险强度
```

### 原则

```text
单一信号不能下结论
多个信号同向才提高可信度
信号冲突时降低仓位或不交易
```

## 五、第三层：选币控制

选币不是找“必涨币”，而是筛掉不适合交易的币。

### 选币过滤条件

```text
成交额足够大
流动性足够好
点差不能过大
波动不能过度异常
资金费率不能极端
近期不能处于剧烈拉盘或砸盘后
```

### 选币评分维度

```text
流动性分
趋势强度分
波动风险分
资金费率风险分
成交量确认分
异常风险分
```

### 输出格式

```text
可交易
只观察
跳过
禁止交易
```

## 六、第四层：方向判断控制

方向判断不能只靠感觉。

### 多头条件示例

```text
趋势偏多
成交量支持上涨
回调不破关键结构
波动没有失控
资金费率不过热
```

### 空头条件示例

```text
趋势偏空
反弹无力
成交量支持下跌
关键支撑失守
资金费率不过度拥挤
```

### 不交易条件

```text
方向冲突
波动过大
流动性差
资金费率极端
连续亏损后
账户处于暂停状态
```

---

## 七、第五层：仓位调节

仓位是控制变量，不是情绪表达。

### 基础规则

```text
普通状态：单笔 ≤ 余额 5%
连续亏损 1 笔：降到 3%
连续亏损 2 笔：降到 2%
连续亏损 3 笔：暂停开仓
高波动状态：仓位减半
低确定性信号：只观察或最小仓位
```

### 控制论解释

```text
亏损反馈 = 系统偏离目标
降低仓位 = 减小控制强度
暂停交易 = 防止系统发散
```

---

## 八、第六层：杠杆调节

杠杆是放大器，也是灵敏度放大器。

### 原则

```text
最大 5x
高波动时 ≤ 3x
连续亏损后 ≤ 2x
信号冲突时不用杠杆或不交易
```

### 控制论解释

```text
杠杆越高，系统对价格扰动越敏感。
灵敏度过高的系统更容易失控。
```

---

## 九、第七层：止损、止盈与最优停止

最优停止回答：什么时候该停。

### 必须定义

```text
入场理由
无效条件
止损位置
止盈条件
最大持仓时间
异常退出条件
```

### 停止条件

```text
达到止损
达到止盈
入场逻辑失效
波动异常扩大
连续亏损触发暂停
账户回撤超过阈值
```

```text
退出理由

```

### 过程控制指标

```text

最大回撤

```

## 十一、第九层：自适应学习

交易系统要根据反馈调节参数。

### 参数调节示例

```text
连续亏损 → 降仓或暂停
胜率下降 → 降低交易频率
盈亏比下降 → 优化止损止盈
高波动亏损多 → 增加波动过滤
震荡行情亏损多 → 增加趋势过滤
```

### 目标

```text
不是让系统越来越激进，而是让系统越来越稳健。
```

## 十三、最重要的一句话

交易系统不是为了预测未来，而是为了在未来不可预测的情况下，让账户不失控。
