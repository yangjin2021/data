## 一、版本定位

本文件用于记录一次新的选品判断补充逻辑，作为 `V1.x` 系列沉淀。

它不直接覆盖主规则文件 `FBA评分体系.txt`，而是作为辅助规则和案例复盘使用。

当前 V1.x 版本结构：

- `V1.0`：综合达成率校准规则。
- `V1.1`：加入门把手选品案例，用于验证校准规则在真实产品判断中的使用方式。

核心原则：

1. 主规则仍然优先执行维度0、真实利润、售后全损、计费重量、原始综合达成率。
2. 校准规则只用于帮助解释“规则过于保守时如何看待产品机会”。
3. 命中否决项、真实毛利润为负、严重侵权、认证/平台限制、运费吞噬利润时，不能通过校准强行变成可做。
4. 校准后综合达成率可以辅助最终展示，但不能覆盖硬性红线。

### 3.5 使用 V1.0 校准规则后的解释

这条商品的原始综合达成率为 `57.96% / 140%`。

由于主规则 version 4.2 的总权重是 140%，而 V1.0 校准规则原本是按 100% 体系设计，所以不能直接把 `57.96%` 当作 100% 制的 57.96%。

如需临时换算：

```yaml
normalized_raw_achievement:
  formula: "57.96 / 140 * 100"
  result: "约41.4%"
```

按 V1.0 校准规则，40%-49% 对应校准后 45%-64%，含义是：

```yaml
calibrated_interpretation:
  normalized_raw_achievement: "约41.4%"
  calibrated_range: "45%-64%"
  judgment: "不建议优先做"
  reason:
    - "机会一般，除非有特殊资源优势。"
    - "该品叠加汽配适配风险、售后全损风险、重量/体积待确认、需求弱等问题。"
    - "即使进行校准，也不能提升到建议做。"
```

## 四、后续使用提醒

以后如果继续采用 V1.x 记录方式，可以这样递增：

- `V1.2`：加入新的产品案例或新的判断逻辑。
- `V1.3`：加入某一类产品的共性风险模板。
- `V1.4`：把多个案例总结为可执行规则。
- 当 V1.x 逻辑经过多次验证后，再考虑合并进主规则 `FBA评分体系.txt`。

建议不要立刻把校准规则写入主规则核心，因为当前主规则已经明确“不输出校准后综合达成率”。如果要合并，需要先决定：

1. 是否恢复“校准后综合达成率”。
2. 140% 权重体系下，校准规则是否要先归一化到 100%。
3. 校准后结果是否只用于展示，而不改变维度0和硬性规则。
