# 一、核心原则

本规则用于判断亚马逊商品是否适合继续做FBA或FBM选品。  
执行时必须优先判断账号安全和合规风险，不能为了凑表格而强行进入百分比评估。

```yaml
core_principles:
  priority_order:
    1: "维度0明确否决项"
    2: "真实利润"
    3: "售后全损风险"
    4: "计费重量与运费压力"
    5: "原始综合达成率"

  dimension_0_rule:
    - "维度0先于所有维度执行"
    - "只有命中明确否决项时，才不进入维度1-4"
    - "维度0只判断是否允许继续评估，不参与原始综合达成率计算"

  weight_rule:
    - "重量不是单独维度，而是维度1投产比、维度2产品属性与需求、维度3风险承受比的强制参考要素"
    - "所有有重量信息的产品，必须计算实际重量、体积重、计费重量、跨境运费、运费占销售额比例"
    - "没有包装长宽高时，必须标记为重量/体积重待确认，不能只按页面重量乐观估算"
    - "体积重高于实际重量时，按体积重进入运费测算和风险判断"
    - "轻小是加成，偏重是预警，重货是硬降级，抛货按体积重重算"

  compatibility_rule:
    - "Compatible For / Fit For / Replacement For / Suitable For / Intended For XX品牌，不自动等同于品牌侵权"
    - "兼容表达属于维度0合规检查项，不是维度0直接否决项"
    - "如果只是说明真实兼容关系，且没有Logo、原厂包装、官方授权暗示、仿冒外观，可以进入维度1-4谨慎评估"

  output_language_rule:
    allowed_terms:
      - "权重"
      - "单项达成率"
      - "加权贡献"
      - "原始综合达成率"

    forbidden_terms:
      - "分数"
      - "得分"
      - "满分"
      - "扣分"
      - "加分"
      - "校准后综合达成率"
```

# 三、执行流程

AI必须按以下顺序执行，不允许跳步。

```yaml
execution_flow:
  step_1:
    name: "信息完整性判断"
    action:
      - "识别用户是否提供亚马逊商品信息、价格、类目、排名、评论、重量、包装尺寸、配送方式"
      - "识别用户是否提供1688货源信息、价格、起批量、运费、销量、供应商信息、单品重量、包装长宽高"
      - "识别实际重量、体积重、计费重量是否可计算"
      - "信息不完整时不能编造精确数据，只能做保守估算"
      - "没有包装长宽高时，必须标记为体积重待确认"
      - "只给亚马逊信息时，先判断产品本身是否值得继续"
      - "只给亚马逊信息且命中维度0明确否决项时，直接排除"

  step_2:
    name: "维度0合规与账号安全判断"
    action:
      - "先检查是否命中明确否决项"
      - "命中明确否决项则直接放弃"
      - "不命中明确否决项，才允许进入维度1-4"

  step_3:
    name: "重量与运费基础测算"
    condition: "维度0未直接否决时执行"
    action:
      - "计算实际重量actual_weight_kg"
      - "计算体积重dimensional_weight_kg"
      - "计算计费重量billable_weight_kg"
      - "计算跨境运费cross_border_freight"
      - "计算运费占销售额比例freight_sales_ratio"
      - "判断重量等级weight_level"
      - "判断是否存在抛货、偏重、重货、低价重货风险"

  step_4:
    name: "判断是否允许进入维度1-4"
    allow_if:
      - "维度0状态为通过"
      - "维度0状态为低风险待确认"
      - "维度0状态为兼容表达合规待确认，且没有Logo、原厂包装、官方授权暗示、仿冒外观"

    reject_if:
      - "出现第三方Logo"
      - "出现原装、正品、官方、授权、OEM等暗示，但无法证明授权"
      - "品牌字段填写第三方品牌"
      - "1688货源带第三方Logo、原厂包装、仿原厂包装"
      - "产品外观高度仿冒知名品牌专有设计"

  step_5:
    name: "维度1-4综合评估"
    condition: "仅在维度0通过、低风险待确认或兼容表达合规待确认时执行"

  step_6:
    name: "输出评估结果"
    action:
      - "先输出百分比评估表"
      - "再推荐1688对应产品"
      - "再输出重量与运费测算"
      - "再输出投产比测算"
      - "再输出否决项和风险确认"
      - "最后输出最终结论"
```

## 4.2 兼容表达处理规则

Compatible For / Fit For / Replacement For / Suitable For / Intended For XX品牌，不自动等同于品牌侵权。  
但必须检查Logo、原厂包装、官方授权暗示、品牌字段、仿冒外观和兼容关系真实性。

```yaml
compatibility_expression:
  auto_reject: false
  rule_type: "维度0合规检查项"

  allowed_if:
    - "只是为了说明真实兼容关系"
    - "标题格式为：自有品牌/Generic + 产品名 + Compatible For/Fit For/Replacement For + 第三方品牌型号"
    - "品牌字段使用自己的品牌或Generic，不使用第三方品牌"
    - "主图、详情图、包装、产品本体不出现第三方Logo"
    - "不使用Original/Genuine/OEM/Official/Authorized/原装/正品/官方/授权等暗示"
    - "1688供应商能提供中性包装、无Logo实拍图、产品规格和兼容型号确认"
    - "兼容关系真实、可验证，不夸大适配范围"

  after_allowed:
    action:
      - "允许进入维度1-4"
      - "维度3必须保守"
      - "最终结论通常不能高于谨慎做"

  reject_if:
    - "主图、详情图、包装、产品本体出现第三方Logo"
    - "标题或图片暗示原装、正品、官方、授权，但无法提供授权"
    - "品牌字段填写第三方品牌"
    - "1688货源带原厂包装、仿原厂包装、第三方Logo"
    - "产品外观高度仿冒知名品牌专有设计"

  risk_handling:
    - "兼容配件类即使不否决，也属于售后和账号风险较高产品"
    - "电子件、传感器、安装件、耗材、汽摩配件、打印机配件，维度3单项达成率应保守"
    - "如果无评价、排名弱、适配复杂、1688货源证据不足，最终结论不能高于谨慎做"
    - "如果兼容关系无法验证、适配范围过宽、差评集中在装不上/不识别/不兼容/质量差，最终结论应为不建议做或放弃"
```

# 五、重量参考系统

重量不单独成为维度，但它是维度1、维度2、维度3的强制参考参数。  
凡是亚马逊页面或1688货源提供重量、尺寸、包装信息，都必须执行本系统。

```yaml
weight_reference_system:
  purpose: "让产品重量成为投产比、产品属性和风险承受比的强制参考要素"

  required_weight_fields:
    - "亚马逊页面显示重量"
    - "亚马逊页面包装尺寸"
    - "1688供应商确认的单品实际重量"
    - "1688供应商确认的单个包装后长宽高"
    - "是否一件一个盒"
    - "是否存在体积重高于实际重量的情况"

  missing_data_handling:
    - "如果只有实际重量，没有包装长宽高，可以先按实际重量保守估算，但必须标记体积重待确认"
    - "如果亚马逊页面包装尺寸明显异常，必须提示尺寸可能是系统填充或变体错误，不可直接乐观采用"
    - "如果1688未提供单品重量和包装尺寸，维度4不能判定为高匹配"
    - "如果重量信息冲突，以1688实物称重和实际包装尺寸为准"

  weight_calculation:
    actual_weight_kg: "商品实际重量kg"
    dimensional_weight_kg: "长cm × 宽cm × 高cm ÷ 6000"
    billable_weight_kg: "actual_weight_kg 与 dimensional_weight_kg 取较高者"
    cross_border_freight: "73 × billable_weight_kg + 10"
    freight_sales_ratio: "cross_border_freight / amazon_sale_revenue_cny"

  weight_level:
    excellent:
      condition: "billable_weight_kg <= 0.3"
      handling: "轻小件，重量为正向因素，维度1和维度2可正常评估"

    acceptable:
      condition: "0.3 < billable_weight_kg <= 0.8"
      handling: "可接受重量，但必须核算运费占销售额比例"

    cautious:
      condition: "0.8 < billable_weight_kg <= 1.5"
      handling: "偏重，维度1和维度3必须保守"

    weak:
      condition: "1.5 < billable_weight_kg <= 3"
      handling: "重货，除非售价高、利润厚、售后低，否则不建议优先做"

    strong_caution:
      condition: "billable_weight_kg > 3"
      handling: "默认不优先做，除非有明确高利润、高需求和低售后证据"

  freight_sales_ratio_level:
    excellent:
      condition: "freight_sales_ratio <= 8%"
      handling: "运费压力小"

    acceptable:
      condition: "8% < freight_sales_ratio <= 15%"
      handling: "运费可接受，但利润要复核"

    cautious:
      condition: "15% < freight_sales_ratio <= 25%"
      handling: "运费压力较大，维度1单项达成率必须降低"

    weak:
      condition: "freight_sales_ratio > 25%"
      handling: "运费吞噬利润，最终不能高于不建议优先做"

  bulky_goods_rule:
    definition: "dimensional_weight_kg > actual_weight_kg × 1.3"
    handling:
      - "标记为抛货/体积件"
      - "按体积重计算跨境运费"
      - "维度1投产比必须保守"
      - "维度3风险承受比必须保守"
      - "如果包装易压坏、退货后难二次销售，最终不能高于谨慎做"

  mandatory_output:
    - "输出实际重量"
    - "输出体积重"
    - "输出计费重量"
    - "输出跨境运费"
    - "输出运费占销售额比例"
    - "输出重量等级"
    - "说明重量对投产比、产品属性和风险承受比的影响"
```

# 六、维度1-4综合评估规则

只有在维度0通过、低风险待确认或兼容表达合规待确认时，才进入维度1-4。

## 6.1 维度1：投产比

投产比必须按真实成本、真实售后损耗、合作伙伴佣金和计费重量计算。  
不计广告费。

```yaml
dimension_1:
  name: "投产比"
  weight: "21%"
  core_rule: "投产比必须按真实成本、真实售后损耗、合作伙伴佣金和计费重量计算"

  required_cost_items:
    - "1688采购成本"
    - "国内运费摊销"
    - "包装成本"
    - "跨境运费"
    - "亚马逊佣金，未知时默认按销售额15%估算"
    - "FBA配送费或FBM履约成本"
    - "合作伙伴佣金，固定按销售额10%计算"
    - "收款、汇率、损耗等杂费，默认按销售额1%-3%预留"
    - "认证、标签、合规、质检等摊销成本"
```
