# 亚马逊FBA选品综合评估规则｜品牌风险强化版

version: "3.0"  
enabled: true  
mode: "账号安全优先 / 保守选品模式"

核心原则:

- "本规则用于判断亚马逊商品是否适合继续做FBA或FBM选品。"
    
- "优先级顺序：维度0否决项 > 真实利润 > 售后全损风险 > 综合达成率。"
    
- "维度0先于所有维度执行。只要维度0命中明确否决项，不进入维度1-4，不输出综合达成率，不推荐1688货源。"
    
- "不要为了凑表格而强行进入百分比评估。"
    
- "统一使用：权重、单项达成率、加权贡献、原始综合达成率、校准后综合达成率。"
    
- "禁止使用：分数、得分、满分、扣分、加分。"
    
- "如果只提供亚马逊商品信息，先判断产品本身是否适合售卖；如果产品本身已命中否决项，直接排除，不要求补1688。"
    
- "如果同时提供亚马逊商品和1688货源信息，仍然必须先执行维度0；维度0通过后才判断投产比、需求、风险和货源匹配。"
    

权重设置:  
weight_mode: "customizable"  
default_weight_input_format: "权重：15/50/15/20"  
total_weight: "100%"

current_weights:  
dimension_1:  
name: "投产比"  
weight: "15%"  
dimension_2:  
name: "产品属性与需求"  
weight: "50%"  
dimension_3:  
name: "风险承受比"  
weight: "15%"  
dimension_4:  
name: "1688货源匹配度"  
weight: "20%"

rule:  
- "用户可以通过修改“权重：15/50/15/20”快速切换维度1-4权重。"  
- "权重顺序固定对应：维度1 / 维度2 / 维度3 / 维度4。"  
- "四个维度权重合计必须为100%。"  
- "维度名称不要随意交换；当前固定为：维度1投产比，维度2产品属性与需求，维度3风险承受比，维度4 1688货源匹配度。"

执行顺序:  
step_1:  
name: "信息完整性判断"  
rule:  
- "识别用户是否提供：亚马逊商品信息、价格、类目、排名、评论、重量、配送方式、1688货源信息。"  
- "信息不完整时不能编造精确数据，只能做保守估算。"  
- "只给亚马逊信息时，先判断产品本身是否值得继续；若命中维度0，直接排除。"

step_2:  
name: "维度0否决项判断"  
rule:  
- "维度0必须最先输出判断结果。"  
- "只要命中明确否决项，直接输出否决项结论表、风险证据表、最终处理建议表。"  
- "命中明确否决项时，不进入维度1-4，不输出原始综合达成率，不输出校准后综合达成率，不推荐1688货源。"  
- "如果是待确认风险，必须判断该风险是否属于账号安全高风险；账号安全高风险默认按经营策略排除。"

step_3:  
name: "是否允许进入维度1-4"  
rule:  
- "只有维度0状态为“通过”或“低风险待确认”时，才允许进入维度1-4。"  
- "如果维度0状态为“品牌/商标/授权高风险”，默认不进入维度1-4。"  
- "如果用户明确表示自己有授权、做专业兼容配件、能处理商标合规，再进入谨慎评估，但必须限制校准后综合达成率上限。"

step_4:  
name: "百分比评估表"  
rule:  
- "维度0通过后，第一部分输出百分比评估表。"  
- "百分比评估表只展示维度1-4。"  
- "每个维度输出：权重、单项达成率、加权贡献、判断。"  
- "加权贡献 = 权重 × 单项达成率。"  
- "原始综合达成率 = 维度1-4加权贡献之和。"

step_5:  
name: "推荐1688对应产品"  
rule:  
- "只有维度0通过后，才推荐1688货源。"  
- "如果维度0命中品牌/侵权/强监管/安全红线，不推荐1688货源，避免误导继续采购。"  
- "推荐1688产品必须标注：截图序号、产品名称、店铺、价格、起批量、运费、销量、匹配理由、风险点。"  
- "如果没有合适货源，必须明确说没有合适货源，不要硬推荐。"

step_6:  
name: "综合达成率校准"  
rule:  
- "保留原始综合达成率，用于解释产品真实短板、风险来源和利润结构。"  
- "输出校准后综合达成率，用于最终决策展示。"  
- "校准后综合达成率必须受否决项、硬性上限、真实利润和一次售后全损影响。"  
- "真实毛利润为负或接近为负时，最终结论不能高于谨慎做。"  
- "一次售后可能吃掉多单利润时，校准后综合达成率最高不超过75%。"  
- "存在重大适配、认证、侵权、平台限制待确认时，校准后综合达成率最高不超过65%。"

维度0否决项:  
name: "维度0：否决项 / 合规红线 / 账号安全红线"  
enabled: true  
position: "先于维度1-4执行，不进入百分比评估表"

核心逻辑:  
- "维度0只判断是否允许继续评估，不参与综合达成率计算。"  
- "命中明确否决项，直接排除。"  
- "待确认风险分为低风险待确认和高风险待确认。"  
- "高风险待确认默认按经营策略排除，除非用户明确具备授权、资质或专业运营能力。"

品牌和知识产权红线:  
name: "品牌 / 商标 / IP / 授权风险"  
default_decision: "保守排除"

```
基础原则:
  - "只要亚马逊标题、主图、详情图、包装、产品本体、品牌字段、五点描述、A+、关键词中出现第三方品牌名、Logo、IP形象或受保护图案，必须先进入维度0判断。"
  - "第三方品牌名包括但不限于：打印机品牌、汽车品牌、摩托车品牌、工具品牌、电子品牌、家电品牌、运动品牌、奢侈品牌、影视动漫游戏IP。"
  - "不要把第三方品牌名当成普通关键词处理。"
  - "不要因为标题里有 for / Compatible with / Fits / Replacement for / Intended for，就自动放行。"
  - "兼容表达在法律或平台层面可能存在合规写法，但在本选品模型中，只要商品需求主要依赖第三方品牌名，默认属于账号安全高风险。"

明确否决:
  - condition: "标题直接出现第三方品牌名，且该品牌名是商品需求核心"
    examples:
      - "Transfer Roller for Lexmark MS821"
      - "Carburetor Kit for Yamaha"
      - "Fuel Cap for Honda"
      - "Sensor for BMW"
      - "Charger for Dyson"
      - "Filter for Shark"
      - "Roller for Brother"
      - "Toner for HP"
      - "Battery for Makita"
    decision: "否决"
    reason: "商品销量依赖第三方品牌适配关系，非授权情况下存在商标投诉、兼容声明错误、图片/包装侵权和账号健康风险。"

  - condition: "主图、详情图、包装、产品本体出现第三方Logo"
    decision: "否决"
    reason: "Logo使用风险高，容易触发商标侵权、假冒、误导性来源投诉。"

  - condition: "标题、图片或文案出现 Original / Genuine / OEM / Official / Authorized / Factory / 原装 / 正品 / 官方 / 授权 等表达，但无法证明授权"
    decision: "否决"
    reason: "暗示原厂、正品或授权，风险极高。"

  - condition: "1688货源图片、包装或产品本体带第三方品牌Logo、原厂包装、仿原厂包装"
    decision: "否决"
    reason: "即使亚马逊竞品页面能上架，也不代表该货源可安全销售。"

  - condition: "1688供应商标题写适用某品牌，但不能提供中性包装、实拍图、无Logo证明、授权或合规说明"
    decision: "否决"
    reason: "货源端不可控，后续上架和售后投诉风险高。"

  - condition: "商品外观、包装、颜色组合、图案、结构高度模仿知名品牌或IP"
    decision: "否决"
    reason: "可能涉及商标、版权、外观专利或商业外观风险。"

  - condition: "标题或图片使用动漫、影视、游戏人物、Logo、角色形象、球队Logo、车标、奢侈品牌图案"
    decision: "否决"
    reason: "无授权时不做。"

兼容表达处理规则:
  rule:
    - "for / Compatible with / Fits / Replacement for / Intended for 不再自动视为安全。"
    - "先判断第三方品牌是否为商品需求核心。"
    - "如果移除第三方品牌名后，买家基本无法搜索或判断产品用途，则说明该品牌是需求核心。"
    - "品牌是需求核心时，默认按经营策略排除。"
    - "如果用户明确有授权、专业做兼容配件、能接受IP投诉风险，才允许进入谨慎评估。"

  可以低风险待确认的极少数情况:
    - "商品是高度通用件，品牌名只是辅助说明，不是核心需求来源。"
    - "标题格式为：自有品牌 + 产品名 + compatible with / for + 第三方品牌型号。"
    - "无Logo、无原厂包装、无官方暗示、无仿冒外观。"
    - "供应商能提供中性包装和实拍图。"
    - "用户明确接受兼容品牌路线风险。"
    - "即使满足以上条件，也不能直接判为建议做，只能进入谨慎评估。"

特别说明:
  - "商品介绍里插件显示的“品牌：XX 复制并检测”，不能单独作为侵权依据。"
  - "但如果XX同时出现在亚马逊标题、图片、包装、五点描述、A+或关键词中，则必须按第三方品牌风险处理。"
  - "竞品已经上架不等于安全。"
  - "销量高不等于安全。"
  - "亚马逊页面没有被下架不等于没有侵权风险。"
  - "1688能找到同款不等于可以安全卖。"
```

name: "风险承受比"

- "带电、精密、重货、易碎、强体验产品，维度3必须保守。"  
- "品牌兼容风险未排除时，不应进入维度3；应在维度0直接处理。"
