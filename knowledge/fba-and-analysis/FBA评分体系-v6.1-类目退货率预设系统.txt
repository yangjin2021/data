# 一、版本定位

本版本不是推翻 v6.0，而是在 v6.0「截图初筛成本闸门最终版」基础上新增一个强制参考模块：

> 产品类目是默认退货率和售后全损预留的重要来源之一。低退货率类目可以适度提高维度1投产比单项达成率；高退货率类目必须提高售后全损预留，并降低维度1投产比单项达成率。

执行时应理解为：

```yaml
execution_stack:
  base_rule: "v6.0 截图初筛成本闸门最终版"
  added_module: "v6.1 类目退货率预设系统"
  if_conflict: "以v6.1新增模块为准，但不改变v6.0的核心流程、维度和权重"
```

# 二、核心原则

```yaml
category_return_rate_core_principles:
  purpose:
    - "将产品类目作为默认退货率和售后全损预留的来源之一"
    - "将类目退货率纳入维度1投产比判断"
    - "让低退货率类目在投产比中获得适度正向修正"
    - "让高退货率类目在投产比中获得强制负向修正"

  important_limits:
    - "类目退货率不是唯一依据，必须同时结合产品重量、易碎性、适配性、安装难度、主观体验、客单价和差评原因"
    - "低退货率类目只能适度提高维度1投产比单项达成率，不能覆盖真实成本、利润闸门、售后全损、账号安全和维度0风险"
    - "高退货率类目必须提高售后全损预留，并降低维度1投产比单项达成率"
    - "如果产品自身存在易碎、重货、安装、尺码、适配、强主观体验问题，即使类目平均退货率低，也不能直接提高投产比"
    - "Subscribe & Save 不是独立商品类目，只代表购买方式；必须结合实际商品所属类目判断"

  calculation_position:
    dimension: "维度1：投产比"
    timing: "截图初筛成本测算之后、利润闸门判断之前或同时执行"
    effect:
      - "影响售后全损预留比例"
      - "影响截图初筛利润"
      - "影响维度1投产比单项达成率"
      - "间接影响最终结论"
```

# 四、按退货率档位调整维度1投产比

```yaml
return_rate_to_roi_adjustment:
  purpose: "把类目退货率映射为售后全损预留和维度1投产比修正"

  levels:
    very_low_return:
      return_rate: "1%-4%"
      after_sales_reserve: "3%"
      dimension_1_roi_adjustment: "+4% to +6%"
      meaning: "极低退货，投产比可适度上调"

    low_return:
      return_rate: "4%-8%"
      after_sales_reserve: "5%"
      dimension_1_roi_adjustment: "+2% to +4%"
      meaning: "低退货，售后压力较小"

    medium_return:
      return_rate: "8%-12%"
      after_sales_reserve: "8%"
      dimension_1_roi_adjustment: "0% to -2%"
      meaning: "中等退货，维持保守"

    medium_high_return:
      return_rate: "12%-18%"
      after_sales_reserve: "12%"
      dimension_1_roi_adjustment: "-3% to -6%"
      meaning: "中高退货，投产比必须下调"

    high_return:
      return_rate: "18%-30%"
      after_sales_reserve: "18%-22%"
      dimension_1_roi_adjustment: "-8% to -15%"
      meaning: "高退货，利润安全垫必须显著提高"

    extreme_high_return:
      return_rate: "30%+"
      after_sales_reserve: "25%+"
      dimension_1_roi_adjustment: "维度1不能高于谨慎区间"
      meaning: "极高退货，默认不优先做"
```

# 五、产品风险倍率

类目退货率只是基础值，必须根据具体产品风险进行倍率修正。

```yaml
product_risk_multiplier:
  standard_simple_product:
    multiplier: "×1.0"
    meaning: "标准简单产品，使用类目默认退货率"

  consumable_or_repeat_purchase:
    multiplier: "×0.7"
    meaning: "消耗品或重复购买品，退货率可适度下调"

  fragile_or_easily_damaged:
    multiplier: "×1.2"
    meaning: "易碎、易压坏、运输损伤风险高"

  size_fit_or_style_sensitive:
    multiplier: "×1.3"
    meaning: "尺码、版型、颜色、审美敏感"

  installation_or_compatibility_sensitive:
    multiplier: "×1.2"
    meaning: "安装、适配、型号匹配风险较高"

  subjective_experience_sensitive:
    multiplier: "×1.2"
    meaning: "香味、手感、音色、舒适度、效果等主观体验强"

  high_value_or_heavy_product:
    multiplier: "×1.2"
    meaning: "高客单、重货、大件，退货处理成本高"
```

# 六、计算规则

```yaml
category_return_rate_calculation:
  step_1:
    name: "读取Amazon类目"
    action: "从Amazon商品截图或文本中识别主类目和细分类目"

  step_2:
    name: "读取类目默认退货率"
    action: "根据default_category_return_rates匹配默认退货率"

  step_3:
    name: "识别产品风险倍率"
    action: "根据易碎、重货、安装、适配、尺码、主观体验、消耗品属性确定倍率"

  step_4:
    name: "计算调整后退货率"
    formula: "adjusted_return_rate = category_default_return_rate_midpoint × product_risk_multiplier"

  step_5:
    name: "计算售后全损预留"
    formula: "after_sales_full_loss_reserve = 单件不可回收成本 × adjusted_return_rate"

  step_6:
    name: "修正维度1投产比"
    action: "根据return_rate_to_roi_adjustment调整维度1投产比单项达成率"
```

# 七、硬性限制

```yaml
category_return_rate_hard_limits:
  - "低退货类目不能覆盖维度0否决项"
  - "低退货类目不能覆盖利润闸门"
  - "低退货类目不能覆盖真实成本亏损"
  - "低退货类目不能覆盖售后全损后亏损"
  - "Clothing, Shoes & Jewelry 默认不得因为售价差大而给出高投产比，除非有明确低退货证据"
  - "Baby 类目即使退货率不是最高，也必须额外考虑安全、材质、认证和差评风险"
  - "Beauty & Personal Care 和 Health & Household 退货率低时可以提高投产比，但不得覆盖合规、功效宣称、皮肤接触、过敏风险"
  - "Subscribe & Save 只代表购买方式，不是独立产品类目；必须结合实际商品所属类目判断"
```
