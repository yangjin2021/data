## 商业分析的最小闭环

以后分析任何生意，至少回答这 8 个问题：

1. **趋势**：这个方向是顺风、平风还是逆风？
2. **客户**：谁最痛、谁最愿意付钱？
3. **痛点**：客户现在最麻烦的事情是什么？
4. **替代方案**：客户现在不用我时，怎么解决？
5. **方案**：我的方案比现有方案好在哪里？
6. **利润**：单个客户、单个订单、单个项目是否赚钱？
7. **增长**：客户怎么来、怎么留、怎么复购、怎么转介绍？
8. **风险**：最坏情况我能不能承受？

## 分析结论统一标签

以后所有商业分析建议统一使用这 5 个结论标签：

| 标签 | 含义 |
|---|---|
| 建议做 | 需求、利润、增长、风险基本成立，可以推进 |
| 谨慎做 | 方向有机会，但关键变量还没验证 |
| 小规模测试 | 不适合重投入，先用 MVP 验证 |
| 暂缓 | 信息不足或条件不成熟，先补数据 |
| 放弃 | 需求、利润、竞争或风险明显不成立 |

## 推荐评分维度

正式分析时，可以按 100% 做综合判断：

| 维度 | 权重 | 说明 |
|---|---:|---|
| 趋势与时机 | 10% | 行业是否顺风，时机是否合适 |
| 客户痛点 | 15% | 需求是否真实、强烈、高频或高价值 |
| 支付意愿 | 10% | 客户是否愿意为结果付钱 |
| 竞争与差异化 | 10% | 是否避开同质化和价格战 |
| 商业模式 | 10% | 收入、成本、交付是否清楚 |
| 单位经济 | 15% | 单客/单单是否真实赚钱 |
| 获客与增长 | 10% | 是否有可行获客路径和增长循环 |
| 交付复制性 | 10% | 是否能标准化、模板化、规模化 |
| 风险与安全边际 | 5% | 最坏情况是否可承受 |
| 护城河与长期资产 | 5% | 是否能沉淀能力、数据、品牌、流程 |

---

## 一句话原则

> 先看趋势，再看竞争；先找客户，再做产品；先算利润，再做增长；先控风险，再重投入；最后把成交沉淀成护城河和飞轮。

---

## 注意

如果仓库是公开仓库，不要把客户隐私、真实合同、身份证、手机号、微信聊天记录、未公开财务数据直接放进来。可以先脱敏，再分析。
