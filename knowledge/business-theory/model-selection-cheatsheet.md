# 商业模型选择速查表

> 用途：当你不知道该用哪个模型时，先看这张表。不要一次性套所有模型，而是按问题选择模型。

---

## 一、按问题选择模型

| 你想解决的问题 | 优先使用模型 | 输出结果 |
|---|---|---|
| 这个行业有没有机会？ | PESTEL、产品生命周期、技术采用生命周期 | 顺风/平风/逆风，机会类型 |
| 这个行业竞争大不大？ | 波特五力、3C、价值链 | 竞争强度、利润空间、进入难度 |
| 我应该服务谁？ | STP、Jobs To Be Done | 目标客户、核心任务、细分定位 |
| 客户痛点是否真实？ | 价值主张画布、用户访谈、JTBD | 痛点强度、替代方案、支付理由 |
| 产品应该先做什么？ | MVP、Kano、RICE、ICE | 最小版本、功能优先级 |
| 这个生意怎么赚钱？ | 商业模式画布、价值定价、订阅制模型 | 收入来源、成本结构、收费方式 |
| 单个订单/客户赚不赚钱？ | 单位经济、价格瀑布、LTV/CAC | 单位利润、回本周期、获客成本 |
| 客户怎么来？ | AARRR、4P、用户旅程地图 | 获客路径、转化路径、留存机制 |
| 如何做增长？ | 北极星指标、OMTM、飞轮模型 | 当前关键指标、增长循环 |
| 会不会被复制？ | VRIO、核心能力、经验曲线 | 护城河、长期资产、复制风险 |
| 现金流安全吗？ | 盈亏平衡、现金转换周期、情景分析 | 回本点、现金压力、悲观情况 |
| 风险能不能承受？ | 敏感性分析、安全边际、决策树、反脆弱 | 最大损失、退出条件、抗风险能力 |
| 执行为什么卡住？ | 约束理论、PDCA、OKR、7S | 瓶颈、改进动作、执行系统 |
| 要不要继续做？ | 综合评分表、关键假设验证、ICE | 建议做/谨慎做/小测/暂缓/放弃 |

## 二、按业务类型选择模型

### 1. AI 产品 / AI 工具 / 私有知识库

优先模型：

1. Jobs To Be Done：客户买的不是 AI，而是某个结果；
2. 价值主张画布：确认是否击中客户痛点；
3. MVP：先做人工版/演示版，不急着完整开发；
4. 单位经济：模型成本、服务器成本、人工交付成本；
5. AARRR：从演示到咨询到成交到续费；
6. VRIO：沉淀模板、行业数据、案例、流程；
7. 跨越鸿沟：从尝鲜用户走向主流客户。

重点问题：

- 客户是否愿意上传真实资料？
- 客户是否愿意为结果付钱，而不是只觉得 AI 很酷？
- 是适合 SaaS，还是项目制交付？
- 交付能不能模板化？

---

### 2. 电商选品 / 淘宝 / 亚马逊 / 抖音带货

优先模型：

1. 产品生命周期：品类是增长还是衰退；
2. 波特五力：竞争和比价是否严重；
3. 单位经济：售价、采购、物流、佣金、广告、售后；
4. 价格瀑布：标价到净利润中间被吃掉多少；
5. 敏感性分析：降价、退货、广告费上升是否还能赚钱；
6. 价值链：供应链、包装、物流、售后哪个环节能优化；
7. BCG：SKU 组合管理。

重点问题：

- 降价 10%-15% 是否还赚钱？
- 一次售后是否吃掉多单利润？
- 是否容易侵权、退货、破损、误购？
- 是否有供应链优势？

---

### 3. 抖音 / 小红书 / 内容获客

优先模型：

1. STP：账号到底服务谁；
2. JTBD：观众为什么看你；
3. AARRR：从观看到关注到私信到成交；
4. 北极星指标：有效咨询数/有效线索数；
5. OMTM：当前阶段只盯一个指标；
6. 飞轮模型：内容、案例、咨询、成交互相增强；
7. 用户旅程地图：从看到视频到付钱的全过程。

重点问题：

- 内容吸引的是目标客户，还是泛流量？
- 用户看完之后下一步做什么？
- 是否有可转化的产品/服务承接？
- 是否能持续生产内容？

---

### 4. 本地服务 / 小生意 / 实体服务

优先模型：

1. STP：锁定区域和人群；
2. 用户旅程地图：客户从发现到复购的路径；
3. 服务蓝图：把服务交付标准化；
4. 单位经济：单次服务利润；
5. RFM：老客户复购与转介绍；
6. 4P：产品、价格、渠道、推广；
7. 现金转换周期：钱什么时候回来。

重点问题：

- 是否有本地高频或高价值需求？
- 是否靠位置、口碑、熟人、短视频、团购获客？
- 服务是否可复制？
- 人工时间是否成为瓶颈？

---

### 5. 咨询服务 / 知识付费 / 个人 IP

优先模型：

1. STP：明确细分客户；
2. JTBD：客户要的不是知识，而是改变结果；
3. 价值定价：按客户收益定价；
4. 服务蓝图：标准化交付；
5. 案例飞轮：案例带来信任，信任带来成交；
6. VRIO：个人经验、表达能力、方法论、案例资产；
7. 订阅制模型：是否能变成长期陪跑或会员。

重点问题：

- 客户买完后能获得什么具体结果？
- 交付是否依赖本人时间？
- 是否能产品化、模板化、课程化？
- 是否能形成案例和口碑？

---

### 6. SaaS / 小程序 / 工具产品

优先模型：

1. JTBD：用户真正任务；
2. Kano：功能取舍；
3. AARRR：增长漏斗；
4. 北极星指标：产品真实价值指标；
5. LTV/CAC：获客是否划算；
6. 订阅制模型：留存和续费；
7. 跨越鸿沟：从早期用户走向主流用户。

重点问题：

- 用户是否高频使用？
- 留存是否足够？
- 获客成本能否被生命周期价值覆盖？
- 是否存在强替代品？

## 三、按阶段选择模型

| 阶段 | 核心任务 | 推荐模型 |
|---|---|---|
| 灵感阶段 | 判断是否值得继续看 | 快速判断卡片、PESTEL、波特五力 |
| 调研阶段 | 找客户和需求 | STP、JTBD、价值主张画布、用户访谈 |
| MVP 阶段 | 最低成本验证 | 精益创业、MVP、ICE、OMTM |
| 首单阶段 | 验证付费 | 价值定价、单位经济、用户旅程地图 |
| 复制阶段 | 标准化交付 | 服务蓝图、PDCA、约束理论、经验曲线 |
| 增长阶段 | 获客和留存 | AARRR、北极星指标、飞轮、4P |
| 放大阶段 | 组织和护城河 | VRIO、核心能力、7S、平衡计分卡 |
| 风险阶段 | 控制损失 | 敏感性分析、情景分析、安全边际、决策树 |

## 四、最小模型组合

### 快速判断组合

适合 10 分钟判断一个想法。

- STP
- JTBD
- 单位经济
- 获客路径
- 风险判断

输出：继续研究 / 小测 / 暂缓 / 放弃。

---

### 标准商业分析组合

适合正式分析一个项目。

- PESTEL
- 波特五力
- STP
- JTBD
- 价值主张画布
- 商业模式画布
- 单位经济
- AARRR
- VRIO
- 敏感性分析

输出：完整商业分析报告。

---

### 重投入前审查组合

适合准备投钱、备货、开发、招人之前。

- 单位经济
- 价格瀑布
- LTV/CAC
- 盈亏平衡
- 现金转换周期
- 敏感性分析
- 情景分析
- 安全边际

输出：是否可以重投入。

---

### 长期业务设计组合

适合判断能否做成长期生意。

- 飞轮模型
- VRIO
- 核心能力
- 经验曲线
- 范围经济
- 动态能力
- 三层增长曲线

输出：护城河和长期资产路线。

## 五、模型使用原则

1. **先用少量模型看清核心问题。**
2. **先客户和利润，再增长和规模。**
3. **没有真实数据时，结论要标注“假设”。**
4. **每次分析最后都必须落到下一步行动。**
5. **所有重投入前都必须做单位经济、敏感性分析和安全边际。**
6. **能小测就不要先重投。**
7. **能人工验证就不要先开发系统。**
8. **能先成交就不要只做调研。**

---

## 六、一句话速查

> 看方向用 PESTEL，看竞争用五力，看客户用 STP 和 JTBD，看产品用 MVP 和 Kano，看赚钱用单位经济，看增长用 AARRR 和飞轮，看护城河用 VRIO，看风险用敏感性分析和安全边际。
