# 可复制商业分析提示词

> 用法：以后你要分析任何项目，直接复制下面的提示词，把【】里的内容换成你的项目信息即可。

---

## 一、完整版商业分析提示词

```text
你是一名商业分析顾问，请基于我提供的项目信息，按照“商业分析工作台”的方法，对这个项目进行系统商业分析。

请不要只套模型名，而是要把模型变成实际判断。分析目标是：判断这个项目是否值得做、怎么做、风险在哪里、下一步如何验证。

【项目信息】
项目名称：
项目一句话描述：
目标客户：
产品/服务内容：
销售渠道：
预估价格：
预估成本：
当前阶段：想法 / MVP / 已有客户 / 已有收入 / 准备放大
已有数据：
我的资源：
我的限制：
我最关心的问题：

请按以下结构输出：

# 项目商业分析报告

## 1. 一句话结论
请直接给出：建议做 / 谨慎做 / 小规模测试 / 暂缓 / 放弃。
并用 3-5 句话说明核心理由。

## 2. 项目基本判断
- 项目本质是什么？
- 它解决了谁的什么问题？
- 它更像产品、服务、工具、平台、内容、渠道，还是资源整合？

## 3. 趋势与大环境分析
使用模型：PESTEL、产品生命周期、技术采用生命周期。
请判断：
- 这个方向是顺风、平风还是逆风？
- 是长期趋势、周期机会、短期热点，还是伪机会？
- 最大外部变量是什么？

## 4. 行业竞争与利润空间
使用模型：波特五力、3C、价值链、蓝海战略。
请判断：
- 竞争强度如何？
- 利润会被谁吃掉？
- 客户是否只看价格？
- 有没有差异化空间？
- 我应该走成本领先、差异化，还是聚焦小众？

## 5. 目标客户与真实需求
使用模型：STP、Jobs To Be Done、价值主张画布、用户旅程地图。
请判断：
- 最适合的目标客户是谁？
- 客户真正想完成什么任务？
- 客户现在的替代方案是什么？
- 客户痛点是否真实、强烈、高频或高价值？
- 客户为什么愿意付钱？

## 6. 产品或服务方案
使用模型：MVP、Kano、服务蓝图、RICE/ICE。
请判断：
- 最小可行产品应该是什么？
- 哪些功能/服务必须做？
- 哪些暂时不要做？
- 客户第一次感受到价值的时刻在哪里？
- 交付流程是否能标准化？

## 7. 商业模式与赚钱逻辑
使用模型：商业模式画布、单位经济、价格瀑布、LTV/CAC、价值定价。
请判断：
- 谁付钱？为什么付钱？
- 收入来源是什么？
- 成本结构是什么？
- 单个客户/订单/项目是否赚钱？
- 获客成本大概会不会过高？
- 是否适合一次性收费、订阅制、服务费、佣金制，还是混合收费？

## 8. 增长路径
使用模型：AARRR、4P、北极星指标、OMTM、飞轮模型。
请判断：
- 客户从哪里来？
- 如何激活？
- 如何留存？
- 如何复购？
- 如何转介绍？
- 当前阶段最应该盯的唯一关键指标是什么？
- 有没有增长飞轮？

## 9. 护城河与长期资产
使用模型：VRIO、核心能力、经验曲线、规模经济、范围经济、动态能力。
请判断：
- 这个项目做久了会沉淀什么？
- 别人为什么不容易复制？
- 如果容易复制，我应该沉淀什么来提高壁垒？
- 是否能形成模板、流程、数据、案例、品牌、客户关系或渠道资产？

## 10. 风险与安全边际
使用模型：敏感性分析、情景分析、安全边际、决策树、反脆弱。
请判断：
- 最大风险是什么？
- 成本上升、售价下降、获客成本上升时还能不能做？
- 最坏情况会损失什么？
- 失败一次是否可承受？
- 什么情况下应该停止？

## 11. 综合评分表
请用百分比输出，不要使用“分数、满分、得分”等词。

| 维度 | 权重 | 单项达成率 | 加权贡献 | 判断 |
|---|---:|---:|---:|---|
| 趋势与时机 | 10% |  |  |  |
| 客户痛点 | 15% |  |  |  |
| 支付意愿 | 10% |  |  |  |
| 竞争与差异化 | 10% |  |  |  |
| 商业模式 | 10% |  |  |  |
| 单位经济 | 15% |  |  |  |
| 获客与增长 | 10% |  |  |  |
| 交付复制性 | 10% |  |  |  |
| 风险与安全边际 | 5% |  |  |  |
| 护城河与长期资产 | 5% |  |  |  |
| 综合达成率 | 100% |  |  |  |

## 12. 关键假设与验证计划
请列出这个项目最需要验证的 5 个假设。
每个假设都要包含：验证方式、最低成本实验、成功标准、失败标准。

## 13. 下一步行动
请给出未来 7 天、30 天、90 天行动计划。
每一步都要具体可执行。

## 14. 最终建议
请明确说明：
- 是否值得继续？
- 是否可以重投入？
- 是否只适合小规模测试？
- 最应该先做的动作是什么？
```

## 二、快速版商业分析提示词

适合临时冒出一个想法时快速判断。

```text
请帮我快速判断这个商业想法是否值得继续研究。

【想法】
一句话描述：
目标客户：
怎么赚钱：
我有什么资源：
我担心的问题：

请按以下结构输出：

1. 一句话结论：建议做 / 谨慎做 / 小规模测试 / 暂缓 / 放弃
2. 客户痛点是否真实？
3. 客户是否愿意付钱？
4. 竞争是否激烈？
5. 单位经济是否可能成立？
6. 获客路径是否清楚？
7. 最大风险是什么？
8. 最小验证动作是什么？
9. 我现在最不该做什么？
10. 下一步 3 个动作。
```

## 三、电商/选品商业分析提示词

```text
请从商业分析角度判断这个电商产品是否值得做。

【产品信息】
产品名称：
目标平台：淘宝 / 抖音 / 亚马逊 / 拼多多 / 其他
目标客户：
售价：
采购成本：
物流成本：
平台佣金：
广告成本：
退货/售后风险：
竞品情况：
供应链情况：

请重点分析：

1. 需求是否真实；
2. 竞争是否过强；
3. 差异化是否存在；
4. 单位经济是否成立；
5. 售后和退货是否会吃掉利润；
6. 获客成本是否可控；
7. 是否适合小规模测试；
8. 最小备货/测试方案；
9. 放弃条件；
10. 最终结论。
```

## 四、AI 产品/工具商业分析提示词

```text
请从商业化角度分析这个 AI 产品/工具是否值得做。

【产品信息】
产品名称：
一句话描述：
目标客户：
客户现在的替代方案：
核心功能：
模型/API/技术成本：
交付方式：SaaS / 私有部署 / 一次性服务 / 咨询交付 / 混合
收费方式：
已有演示或案例：

请重点分析：

1. 客户买的是 AI，还是买某个具体结果？
2. 真实任务 Jobs To Be Done 是什么？
3. 是否存在高频或高价值场景？
4. 客户是否愿意上传真实资料/数据？
5. 模型成本、服务器成本、人工交付成本是否可控？
6. 是适合标准化 SaaS，还是适合项目制交付？
7. 如何设计 MVP？
8. 如何做抖音/私域演示获客？
9. 能沉淀什么护城河？
10. 最终结论和下一步验证计划。
```

## 五、本地服务/小生意商业分析提示词

```text
请分析这个本地服务/小生意是否值得做。

【项目信息】
服务内容：
所在城市/区域：
目标客户：
单次收费：
主要成本：
获客方式：
竞争对手：
交付流程：
我能投入的时间和资金：

请重点分析：

1. 本地需求是否真实；
2. 客户是否有急迫痛点；
3. 是否依赖熟人/地推/抖音/团购平台；
4. 单次利润是否足够；
5. 是否能复购或转介绍；
6. 服务是否容易标准化；
7. 最大售后风险是什么；
8. 最小测试方式；
9. 是否适合一个人先做；
10. 最终结论。
```

## 六、输出风格要求

以后使用这些提示词时，建议加上：

```text
请直接、务实、偏保守地分析。不要为了鼓励我而美化项目。看到明显不成立的地方要直接指出。所有结论都要落到下一步行动，不要只讲理论。
```
