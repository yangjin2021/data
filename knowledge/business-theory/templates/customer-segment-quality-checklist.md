# 目标客户人群质量判断清单

> 用途：判断一个人群是不是适合当前阶段去服务、获客、销售和交付。这里的“好人群 / 坏人群”不是评价人本身，而是判断这个人群是否适合当前项目切入。

---

## 一、核心判断原则

一个好人群通常不是“人数最多的人群”，也不是“听起来最高端的人群”，而是同时满足：

> 有真实痛点、有资料或需求基础、有付费能力、有决策权、能被触达、你当前能交付。

对早期个人创业者来说，尤其要避免一开始服务：

- 只好奇、不行动的人；
- 没有预算、只想免费试的人；
- 没有资料或没有真实业务场景的人；
- 决策链条太长的人；
- 合规、责任、定制要求过高的人；
- 需求无限扩大、售后边界不清的人。

---

## 二、判断好人群的 8 个标准

| 维度 | 好人群特征 | 坏人群特征 | 判断问题 |
|---|---|---|---|
| 1. 痛点强度 | 现在真的被问题困扰，问题高频、烦人、影响收入或效率 | 只是觉得新鲜、好玩、可有可无 | 这个问题现在是否正在让他损失时间、钱、效率或机会？ |
| 2. 资料/需求基础 | 手里已经有资料、流程、客户问题、业务场景 | 没资料、没业务场景、只是想看看 | 他有没有能立刻拿来测试的真实资料或真实任务？ |
| 3. 付费能力 | 愿意为省时间、省人工、提高效率、减少错误付钱 | 只想免费白嫖，价格极度敏感 | 他是否能接受测试费、部署费、服务费或订阅费？ |
| 4. 决策速度 | 一个老板、负责人或个人就能拍板 | 要层层审批、采购、法务、IT、安全审查 | 谁能决定买？多久能决定？ |
| 5. 使用频率 | 每周甚至每天都会用，能形成习惯 | 偶尔用一次，没有持续场景 | 这个工具或服务会不会反复被使用？ |
| 6. 结果可见 | 用完马上能看到效果，例如省时间、生成内容、减少重复问答 | 效果抽象，很难证明 | 第一次交付后，客户能不能马上感知价值？ |
| 7. 交付难度 | 你能远程、低成本、标准化交付 | 需要复杂定制、强合规、长期驻场、高责任承诺 | 以你当前能力，能不能顺利交付并控制售后？ |
| 8. 复购/转介绍 | 用好了能继续维护、升级、买模板、介绍别人 | 一次性尝鲜，没有后续价值 | 他用好以后，是否有续费、复购、升级或转介绍可能？ |

## 三、快速评分表

每个维度按 1-5 分打分：

- 1 分：很弱 / 不成立；
- 2 分：偏弱；
- 3 分：一般；
- 4 分：较好；
- 5 分：很强。

| 维度 | 权重 | 评分 1-5 | 加权判断 |
|---|---:|---:|---|
| 痛点强度 | 20% |  |  |
| 付费能力 | 15% |  |  |
| 决策速度 | 15% |  |  |
| 资料/需求基础 | 15% |  |  |
| 使用频率 | 10% |  |  |
| 交付难度 | 10% |  |  |
| 获客难度 | 10% |  |  |
| 复购/转介绍潜力 | 5% |  |  |

### 判断标签

| 综合判断 | 含义 | 行动 |
|---|---|---|
| 80 分以上 | 优质人群 | 可以优先测试和成交 |
| 65-80 分 | 可测试人群 | 先做小样本验证 |
| 50-65 分 | 谨慎人群 | 只适合低成本接触，不重投入 |
| 50 分以下 | 暂不适合 | 先放弃或后期再看 |

## 四、好客户与坏客户的语言信号

### 好客户常说

| 客户说的话 | 代表什么 |
|---|---|
| “我这里确实一堆资料没人看。” | 有资料基础和真实痛点 |
| “员工老是问重复问题。” | 高频需求 |
| “这个能不能放我自己的电脑或服务器？” | 有隐私意识，也可能有付费意愿 |
| “能不能帮我装好？” | 愿意买交付，不只是买源码 |
| “多少钱？” | 有购买意向 |
| “我发你一份资料试试。” | 愿意进入验证阶段 |
| “这个能不能给我员工/学员/客户用？” | 有真实业务场景 |

### 坏客户常说

| 客户说的话 | 风险信号 |
|---|---|
| “有没有免费的？” | 付费意愿弱 |
| “你把源码发我，我自己研究。” | 可能不买服务，容易只看技术 |
| “我还没资料，先看看。” | 产品价值很难体现 |
| “能不能什么都支持？” | 需求泛化，容易失控 |
| “你能保证 100% 准确吗？” | 责任风险高 |
| “我们公司要走采购流程。” | 周期长，早期不适合 |
| “先免费帮我做完整套看看。” | 白嫖和售后消耗风险高 |

## 五、适合 AI 私有资料库 / 企业资料 AI 老员工项目的人群判断

### 好人群画像

| 人群 | 为什么适合 |
|---|---|
| 小公司老板 | 有制度、产品资料、客服问题、销售话术，员工经常重复问 |
| 培训老师 / 课程博主 | 有课程、讲义、题库、学员问答，适合做 AI 助教 |
| 客服/销售团队负责人 | 有大量重复问题，需要统一回复口径 |
| 跨境电商小团队 | 有产品、竞品、采购、规则资料，需要整理和分析 |
| 咨询顾问 / 自媒体资料型创业者 | 资料多，需要转化成内容、方案和交付物 |
| 门店老板 / 服务行业老板 | 有服务流程、报价、客户问题、新人培训需求 |

### 暂时不适合的人群

| 人群 | 原因 |
|---|---|
| 学生个人 | 需求可能多，但付费能力弱，容易白嫖 |
| 纯 AI 爱好者 | 只想试工具，不一定买服务 |
| 大企业 | 有钱但流程长，采购、法务、安全、IT 要求高 |
| 医疗/金融/法律专业机构 | 准确性、合规和责任风险过高，早期不要碰 |
| 没资料的人 | 没有资料可上传，产品价值出不来 |
| 只想买源码的人 | 不懂的人不会用，懂的人不需要你 |

## 六、找客户时的 5 个筛选问题

1. 你们公司/课程/业务有没有一堆 PDF、Word、产品资料、制度资料或常见问题？
2. 员工、客户、学员会不会经常问重复问题？
3. 这些问题现在是谁在回答？会不会占用你很多时间？
4. 如果有个 AI 能根据你的资料回答问题，你愿不愿意发一份资料试试？
5. 如果测试效果好，几百到一两千帮你部署好，能接受吗？

如果对方同时满足：

- 有资料；
- 有重复问题；
- 能拍板；
- 愿意发资料测试；
- 能接受小额测试费或部署费；

就可以作为早期重点跟进客户。

## 七、可查的数据来源网站

判断一个人群是否值得做，可以优先查这些数据来源：

| 数据类型 | 推荐来源 | 看什么 |
|---|---|---|
| 中国企业数量、个体户、行业分布 | 国家统计局、全国经济普查公报 | 某类企业/行业数量是否足够大 |
| 中小企业数量与政策 | 工信部、中国政府网、中小企业公共服务平台 | 中小企业规模、政策、服务需求 |
| 全球 SME 数据 | World Bank SME Finance、OECD SME Digitalisation | 中小企业占比、融资/数字化/技能缺口 |
| AI 工作场景采用 | Microsoft Work Trend Index、Salesforce SMB AI reports | AI 使用率、组织落地难点、SMB 采用情况 |
| 行业趋势与关键词 | 百度指数、微信指数、巨量算数、抖音热点宝、Google Trends | 客户是否在搜索、讨论、关注这个问题 |
| 企业获客线索 | 企查查、天眼查、爱企查、Boss 直聘、招聘平台 | 某行业公司数量、岗位需求、组织规模 |
| 内容需求 | 小红书、抖音、知乎、B 站、公众号 | 用户痛点、评论区问题、内容热度 |
| 海外 SaaS/软件趋势 | G2、Capterra、Product Hunt、Similarweb | 同类产品是否有人买、用户吐槽什么 |

## 八、一句话口诀

> 有资料、有重复、有钱、有权、你能搞定 = 好人群。  
> 没资料、没预算、只好奇、要定制、风险高 = 暂不适合的人群。

更简单地说：

> 好客户买结果，坏客户问功能。  
> 好客户想省事，坏客户想白嫖。  
> 好客户有真实资料，坏客户只有兴趣。
